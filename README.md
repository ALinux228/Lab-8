## Каталог музыки

Разработать консольное приложение с дружественным интерфейсом с возможностью выбора заданий для работы с «базой данных (БД)», хранящейся в бинарном файле. Перечень полей (минимум 5), достаточно полно характеризующих заданную в варианте предметную область, предложить самостоятельно (постараться отразить в перечне полей такие, которые требуют разных типов данных). В классе должны присутствовать свойства, конструкторы, перегруженный метод ToString(). Весь функционал приложения реализовать в виде методов вспомогательного класса с помощью LINQ-запросов. Предусмотреть обработку возможных ошибок при работе программы.


## Задача 1

### Чтение базы данных из бинарного файла

Реализовать чтение БД из бинарного файла в виде метода класса.

### Алгоритм решения

Проверить, существует ли файл.
   
Если файл существует:

  1. Открыть файл
  
  2. Создать объект для чтения бинарных данных
  
  3. Очистить существующий список
  
  4. Прочитать количество записей (первые 4 байта файла)
  
  5. В цикле for прочитать каждую запись:
  
    1. Читать каждое поле строго соблюдая порядок заполнение
    2. Создать объект MusicItem и заполнить его поля
    3. Добавить объект в список items
    4. Потоки закрываются автоматически через using
    
Если файл не существует:

  1. Инициализировать пустой список items
  
  2. Вызвать метод для создания нового бинарного файла

2. Запросить у пользователя ID для удаления, выполнить парсинг.

5. Если найден - запросить подтверждение.
    
6. При подтверждении вызвать метод, который удаляет элемент из списка и вызывает метод для обновления бинарного файла.

## Задача 5

### Запросы

Реализация 4 запросов с помощью LINQ:
1. (Перечень) Показать альбомы дороже заданной цены.
2. (Перечень) Показать альбомы с рейтингом выше заданного.
3. (Одно значение) Показать общее количество прослушиваний всех альбомов.
4. (Одно значение) Показать среднюю цену альбома.

### Алгоритм решения

Запрос 1 (перечень) - GetAlbumsExpensive(decimal minPrice)
- Выполняется LINQ-запрос: запрос выбирает из списка items (коллекция всех альбомов) только те альбомы, у которых цена (Price) больше заданной минимальной цены (minPrice). Затем полученные альбомы сортируются в порядке убывания цены (от самого дорогого к самому дешёвому). Результат запроса преобразуется в список (ToList()), который возвращается вызывающему коду.
- Возвращает список альбомов, отсортированных по убыванию цены.

Запрос 2 (перечень) - GetAlbumsHighRaiting(double minRating)
- LINQ-запрос: запрос выбирает из списка items только те альбомы, у которых рейтинг (Rating) больше заданного минимального значения (minRating). Отобранные альбомы сортируются в порядке убывания рейтинга (сначала самые высокооценённые). Результат возвращается в виде списка (ToList())
- Возвращает список альбомов, отсортированных по убыванию рейтинга.

Запрос 3 (одно значение) - GetTotalListenCount()
- LINQ-запрос: запрос проходит по всем альбомам в списке items и суммирует значения поля ListenCount (количество прослушиваний) для каждого альбома. В результате возвращается одно число – общее количество прослушиваний всех альбомов в базе данных.
- Возвращает суммарное количество прослушиваний.

Запрос 4 (одно значение) - GetAveragePrice()
- LINQ-запрос: запрос вычисляет среднее арифметическое цен всех альбомов в списке items. Для этого он складывает цены всех альбомов и делит на их количество. Если в списке нет ни одного альбома, запрос возвращает 0 (чтобы избежать ошибки деления на ноль).
- Возвращает среднюю цену всех альбомов (если список пуст - возвращает 0).
